## Migration Guide

Follow these steps to upgrade yangheIDE smoothly.

<Steps>
  <Step title="Backup Config" icon="database">

    Save your current settings:

````bash
cp -r ~/.config/yangheide ~/.config/yangheide-backup
````

  </Step>
  <Step title="Download Release" icon="download">

    Get the latest binary from releases.

  </Step>
  <Step title="Install" icon="package">

    <Tabs>
      <Tab title="macOS" icon="apple">

````bash
brew install --cask yangheide
````

      </Tab>
      <Tab title="Linux" icon="linux">

````bash
sudo dpkg -i yangheide_1.2.0_amd64.deb
````

      </Tab>
    </Tabs>

  </Step>
  <Step title="Migrate Config" icon="settings">

    Update to TOML if from v0.x:

````toml
# config.toml
theme = "dark"
keybind = "vim"
````

  </Step>
</Steps>

<Callout kind="tip">
  Test in a new workspace first. Run `yangheide --version` to verify installation.
</Callout>

<Expandable title="Deprecations" default-open="false">

In v1.2.0:

- Legacy JSON config deprecated (remove by v2.0)
- `yangheide --debug` flag replaced with `yangheide --trace`

Migration example:

````bash
# Old JSON
mv ~/.config/yangheide/config.json ~/.config/yangheide/config.json.bak

# New TOML (auto-generated on first run)
yangheide --migrate-config
````

</Expandable>
